# Research

Use this tree for evidence gathering, comparisons, and reference-driven notes.

This is not a copy-tracking folder. It is where `zbar` records how it learned
from strong existing implementations and adapted those lessons into its own
architecture.

## Required For Difficult Issues

If an issue is hard enough that a wrong memory-based guess would waste time or
risk architecture drift, write a research note here.

The note should include:

- question being answered
- reference repo(s) inspected
- file/path citations inside those repos
- conclusion taken into `zbar`
- any deliberate difference from the reference design

Current local reference repo setup is documented in:

- [Reference Repos](../reference_repos.md)
